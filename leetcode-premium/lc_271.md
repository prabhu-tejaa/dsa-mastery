# LC 271 — Encode and Decode Strings

**Difficulty:** Medium

---

## Problem Statement

Design an algorithm to encode a list of strings to a single string. The encoded string is then sent over the network and decoded back to the original list of strings.

**Machine 1 (sender):**
```
    public String encode(List<String> strs) {
        // your code
    }
```

**Machine 2 (receiver):**
```
    public List<String> decode(String s) {
        // your code
    }
```

---

## Examples

**Example 1 — Basic case:**
```
Input:  ["Hello", "World"]
Encoded: "Hello#World"
Decode: split by # → ["Hello", "World"] ✅
```

**Example 2 — Why simple delimiter breaks:**
```
Input:  ["He#llo", "Wor#ld"]
Encoded: "He#llo#Wor#ld"
Decode: split by # → ["He", "llo", "Wor", "ld"] ❌ WRONG
```

**Example 3 — Empty strings:**
```
Input:  ["", "hello", ""]
Encoded: "#hello#"
Decode: split by # → ??? ❌ totally broken
```

**The real challenge:** How do you encode such that the decoder always knows
exactly where one string ends and the next begins,
no matter what characters are inside the strings?

---

## Constraints

- `0 <= strs.length < 100`
- `0 <= strs[i].length < 200`
- `strs[i]` contains any possible characters out of 256 valid ASCII characters

---

## Follow Up

Could you write a generalized algorithm to work on any possible set of characters?
